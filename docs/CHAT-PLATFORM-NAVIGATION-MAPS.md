# LLM Chat Platform Navigation Maps: A Complete Structural Analysis

**Five major AI chat platforms reveal strikingly different approaches to user experience architecture.** While all share the fundamental sidebar-plus-chat pattern, their navigation flows, page structures, and URL hierarchies diverge significantlyâ€”from ChatGPT's "try before you sign up" approach to DeepSeek's completely free model with no pricing pages. This analysis maps every user-facing page, navigation path, and layout element across Claude, ChatGPT, Gemini, Grok, and DeepSeek.

## Claude: Marketing and app live on separate domains

Anthropic maintains a clear separation between its **marketing domain** (claude.com) and **application domain** (claude.ai). Visiting claude.ai without authentication immediately redirects to the login pageâ€”there's no guest access or trial period.

### Page structure and URLs

| Page           | URL                             |
| -------------- | ------------------------------- |
| Marketing home | `claude.com`                    |
| App/Chat       | `claude.ai`                     |
| Login/Signup   | `claude.ai/login`               |
| Pricing        | `claude.com/pricing`            |
| Team pricing   | `claude.com/pricing/team`       |
| Enterprise     | `claude.com/pricing/enterprise` |
| Help center    | `support.claude.com`            |

**Login and signup are combined** on a single page at `/login`. Claude uses a **passwordless authentication model**â€”users sign in via Google OAuth, Apple OAuth, or email magic links (one-time secure links sent to their inbox). No traditional password system exists.

## ChatGPT: Try before signup with unified app experience

ChatGPT takes a fundamentally different approach: **visitors can start chatting immediately** without creating an account. The main domain (chatgpt.com) serves as both landing and applicationâ€”there's no separate marketing site at that domain.

### Page structure and URLs

| Page               | URL                                     |
| ------------------ | --------------------------------------- |
| Main chat          | `chatgpt.com`                           |
| Login/Signup       | `chatgpt.com/auth/login`                |
| Pricing            | `chatgpt.com/pricing`                   |
| GPT Store          | `chatgpt.com/gpts`                      |
| Individual chat    | `chatgpt.com/c/[conversation-id]`       |
| Marketing overview | `openai.com/chatgpt/` (separate domain) |

OpenAI consolidated multiple domains: **chat.openai.com** and **chat.com** both redirect to chatgpt.com. Login and signup use **combined pages** with Google, Microsoft, and Apple SSO options alongside email/password.

## Key structural patterns and differences

**Guest access varies dramatically.** ChatGPT allows immediate use without signup (single session, no history). Claude, Gemini, and DeepSeek require authentication before any interaction. Grok offers limited queries before requiring signup.

**Marketing separation follows two models.** Claude, Gemini, and DeepSeek maintain strict separation between marketing and application domains. ChatGPT and Grok blur this lineâ€”chatgpt.com is primarily the app, and grok.com serves both purposes.

**Authentication approaches diverge significantly.** Claude is fully passwordless (magic links only). Gemini requires Google accounts exclusively. ChatGPT, Grok, and DeepSeek offer traditional email/password alongside social OAuth.

**Settings have no dedicated URLs across all platforms.** Every platform handles settings through in-app modals or panels, accessed via profile menus. Only Grok offers direct URLs for specific functions like `/history` and `/plans`.

**Pricing integration varies by business model.** DeepSeek has no consumer pricing pages (completely free). Claude separates pricing to the marketing domain. ChatGPT, Gemini, and Grok integrate pricing access within the app interface through profile/settings menus.
